Требования к системе прогнозирования оттока клиентов (E-commerce)

Бизнес-контекст
Задача: прогнозировать, какие клиенты перестанут совершать покупки в течение следующих 3 месяцев.

	Бизнес-требования
1. Precision ≥ 0.70 — минимизация ложных срабатываний (чтобы не беспокоить лояльных клиентов)
2. Recall ≥ 0.65 — выявление большинства потенциальных уходов
3. Время инференса < 100 мс на одного клиента
4. Интерпретируемость — модель должна объяснять предсказания (feature importance, SHAP)
5. Справедливость — отсутствие дискриминации по полу, возрасту и другим защищённым признакам

	ML-метрики
Основные: Precision, Recall, F1-score
Дополнительные: ROC-AUC, PR-AUC, Confusion Matrix
Бизнес-метрика: Экономический эффект (Cost-Benefit Analysis)

	Функциональные требования
1. Загрузка данных из PostgreSQL
2. Предобработка данных (очистка пропусков, кодирование категорий)
3. Обучение модели на исторических данных
4. Прогнозирование оттока для новых клиентов
5. REST API для инференса
6. Логирование предсказаний и мониторинг дрейфа данных

	Нефункциональные требования
1. Производительность: Обучение модели < 10 минут, инференс < 100 мс
2. Надёжность: Доступность API 99.9%
3. Масштабируемость: Поддержка до 1000 RPS
4. Безопасность: Аутентификация для API (опционально)

	Данные
Источник: E-commerce датасет (5630 клиентов, 20 признаков)
Целевая переменная: Chur` (1 — ушёл, 0 — остался)
Признаки: 
	Демографические (Gender, CityTier, MaritalStatus)
	Поведенческие (Tenure, HourSpendOnApp, OrderCount, CouponUsed)
	Транзакционные (CashbackAmount, OrderAmountHikeFromlastYear)
	Сервисные (Complain, SatisfactionScore)

	Ограничения
1. Пропуски в данных: 250–300 значений в числовых признаках
2. Дисбаланс классов: ~16.5% клиентов ушли (предварительно)
3. Регуляторные требования: GDPR, объяснимость модели

	План экспериментов
1. Baseline: Логистическая регрессия
2. Сравнение: Random Forest, Gradient Boosting, XGBoost
3. Гиперпараметрическая оптимизация: GridSearchCV/RandomizedSearchCV
4. Валидация: Stratified K-Fold (k=5)
5. A/B тест: Сравнение с текущей системой (если есть)
